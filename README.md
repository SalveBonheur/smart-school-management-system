### Prerequisites
- Node.js (v14 or higher)
- MySQL database server
- npm or yarn package manager

### Setup Instructions

1. **Clone the repository:**
   ```bash
   git clone <repository-url>
   cd smart-school-transport-system
   ```

2. **Install dependencies:**
   ```bash
   npm install
   ```

3. **Database Setup:**
   - Import the `database/schema.sql` file into your MySQL database
   - Run the `database/setup.js` script to initialize data
   ```bash
   node database/setup.js
   ```

4. **Environment Configuration:**
   Create a `.env` file in the root directory:
   ```env
   PORT=3000
   NODE_ENV=development
   DB_HOST=localhost
   DB_USER=your_username
   DB_PASSWORD=your_password
   DB_NAME=school_transport
   JWT_SECRET=your_jwt_secret_key
   LOG_LEVEL=info
   ```

5. **Start the application:**
   ```bash
   npm start
   ```

6. **Access the application:**
   - Admin Panel: http://localhost:3000/admin
   - Professional Admin Panel: http://localhost:3000/admin-pro
   - API Health Check: http://localhost:3000/api/health

## 🔧 API Endpoints

### Authentication
- `POST /api/auth/login` - User login
- `POST /api/auth/register` - Admin registration
- `GET /api/auth/profile` - Get user profile

### Students
- `GET /api/students` - Get all students with pagination
- `POST /api/students` - Create new student
- `PUT /api/students/:id` - Update student
- `DELETE /api/students/:id` - Delete student
- `GET /api/students/:id` - Get student by ID

### Buses
- `GET /api/buses` - Get all buses
- `POST /api/buses` - Create new bus
- `PUT /api/buses/:id` - Update bus
- `DELETE /api/buses/:id` - Delete bus

### Drivers
- `GET /api/drivers` - Get all drivers
- `POST /api/drivers` - Create new driver
- `PUT /api/drivers/:id` - Update driver
- `DELETE /api/drivers/:id` - Delete driver

### Routes
- `GET /api/routes` - Get all routes
- `POST /api/routes` - Create new route
- `PUT /api/routes/:id` - Update route
- `DELETE /api/routes/:id` - Delete route

### Attendance
- `GET /api/attendance` - Get attendance records
- `POST /api/attendance` - Record attendance
- `GET /api/attendance/student/:id` - Get student attendance
- `GET /api/attendance/reports/date-range` - Generate attendance reports

### Payments
- `GET /api/payments` - Get payment history
- `POST /api/payments` - Record payment
- `GET /api/payments/student/:id` - Get student payments

### Dashboard
- `GET /api/dashboard/stats` - Get system statistics
- `GET /api/dashboard/charts` - Get chart data
- `GET /api/dashboard/alerts` - Get system alerts

## 🚀 Deployment

### Production Deployment
1. **Environment Setup:**
   ```bash
   # Set production environment variables
   NODE_ENV=production
   PORT=3000
   # ... other production configs
   ```

2. **Build Process:**
   ```bash
   # Install production dependencies
   npm install --production
   
   # Run database migrations
   node database/setup.js
   ```

3. **Process Management:**
   ```bash
   # Using PM2 for production
   npm install -g pm2
   pm2 start server.js --name "school-transport"
   pm2 startup
   pm2 save
   ```

4. **Reverse Proxy (Nginx):**
   ```nginx
   server {
       listen 80;
       server_name your-domain.com;
       
       location / {
           proxy_pass http://localhost:3000;
           proxy_http_version 1.1;
           proxy_set_header Upgrade $http_upgrade;
           proxy_set_header Connection 'upgrade';
           proxy_set_header Host $host;
           proxy_set_header X-Real-IP $remote_addr;
           proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
           proxy_set_header X-Forwarded-Proto $scheme;
           proxy_cache_bypass $http_upgrade;
       }
   }
   ```

### Docker Deployment
```dockerfile
# Dockerfile
FROM node:16-alpine
WORKDIR /app
COPY package*.json ./
RUN npm install --production
COPY . .
EXPOSE 3000
CMD ["npm", "start"]
```

```bash
# Build and run
docker build -t school-transport .
docker run -p 3000:3000 school-transport
```
